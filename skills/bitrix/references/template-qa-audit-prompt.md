# Template: QA Audit Prompt (Static + Dynamic, A-I)

Use this prompt as a strict QA task starter.

```text
Ты QA-инженер. Проверь модуль <module_id> по чек-листу:
A Установка/Удаление/Обновление
B Качество кода (локализация, магические числа, debug)
C Сквозные сценарии (описать 5–10 главных)
D Производительность и масштабирование
E UX на больших данных
F Безопасность (права, CSRF, path traversal)
G Надёжность (locks, возобновление, параллельность)
H Диагностика/логи
I Совместимость

Для каждого пункта выдай:
- PASS/FAIL/N-A
- доказательство (команда/скрин/лог/SQL)
- риск (low/med/high)
- конкретная правка (что/где/как)

Сначала сделай статический аудит кода (grep/поиск строк/чисел), затем динамические тесты в админке и CLI.
В конце сформируй единый отчёт и список задач для исправления, отсортированный по риску.
```

## Optional strict-output appendix

```text
Формат итогового отчёта:
1) Summary table A-I
2) Detailed FAIL items
3) Risk-ranked fix backlog (high -> med -> low)
4) Re-test plan
```
